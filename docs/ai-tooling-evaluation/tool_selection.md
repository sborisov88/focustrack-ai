# Обоснование выбора AI-инструмента

## Выбранный инструмент

**Codex** — основной AI coding assistant для разработки FocusTrack AI.

## Ключевые факторы решения

1. **Паритет возможностей с Cursor.** Codex и Cursor закрывают один и тот же класс задач: локальный репозиторий, git, терминал, многофайловые правки, агентный цикл и работа с контекстом проекта.
2. **Стоимость владения.** По OpenAI Codex pricing Codex входит в планы ChatGPT, включая Free/Go/Plus/Pro/Business/Edu/Enterprise. Для текущего проекта это означает работу через уже используемую подписку.
3. **Отсутствие второй подписки.** Cursor — сильная альтернатива, но по Cursor pricing это отдельный продукт с собственными планами, включённым объёмом model usage и оплатой on-demand usage после исчерпания включённого объёма.
4. **Предсказуемость бюджета.** Для проекта на несколько месяцев важнее не искать минимальные отличия в функциях, а не добавлять отдельную платную IDE-среду, если уже доступен сопоставимый агент.
5. **Практическая достаточность.** Codex уже покрывает нужный engineering workflow: локальные файлы, проверки, `git diff`, `git status`, коммиты и push в GitHub.

## Сравнение с альтернативами

### Cursor

**Плюсы:** сильная AI-first IDE, agentic development, rules/plugins, хороший локальный контекст. По ключевым возможностям находится на одном уровне с Codex.

**Минусы:** отдельная IDE-подписка. Даже если функционально Cursor сопоставим с Codex, для текущего проекта он добавляет отдельный платёжный контур.

**Вердикт:** сильная альтернатива, но не основной инструмент для текущего проекта из-за стоимости владения.

### Bolt.new, Lovable и Replit Agent

**Плюсы:** быстрый первый визуальный результат, удобны для прототипов.

**Минусы:** cloud-first workflow, слабее контроль над локальной структурой репозитория, git-историей, проверками и будущей backend/security-частью.

**Вердикт:** полезны для UI-идей и быстрых прототипов, но не как основной инструмент разработки FocusTrack AI.

### GitHub Copilot

**Плюсы:** хорошая интеграция с GitHub и IDE, сильное автодополнение, agentic features.

**Минусы:** менее удобен как единая среда “правка → проверка → push” для текущего локального проекта.

**Вердикт:** можно использовать дополнительно, но не основной выбор.

### Cline

**Плюсы:** open-source agent runtime, Plan/Act, MCP, editor/terminal workflow.

**Минусы:** больше ручной настройки модели, бюджета и рабочего процесса.

**Вердикт:** сильная техническая альтернатива, но требует больше самостоятельной настройки модели и бюджета.

## Связь с LLM-моделями для будущего AI-слоя

GigaCode, YandexGPT, DeepSeek и Qwen важны для будущего выбора модели в AI-слое FocusTrack AI. Но здесь выбирается именно рабочий **AI coding assistant** для разработки проекта. Поэтому Codex выбран как основной инструмент разработки, а модели будут оцениваться отдельно при реализации Edge Functions или другого backend-слоя.

## Вывод

Codex выбран не потому, что радикально превосходит Cursor по возможностям. По ключевым инженерным функциям они сопоставимы. Главный аргумент — стоимость: Codex уже включён в подписку ChatGPT и позволяет не оплачивать отдельную IDE-подписку для сопоставимого рабочего процесса.
