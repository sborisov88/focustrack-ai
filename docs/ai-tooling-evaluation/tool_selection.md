# Обоснование выбора AI-инструмента

## Выбранный инструмент

**Codex** — основной AI coding assistant для разработки FocusTrack AI.

## Ключевые факторы решения

1. **Рабочий процесс.** Проект ведётся в Codex: редактирование файлов, проверки, коммиты и push в GitHub.
2. **Локальный контекст.** Codex работает с локальной директорией проекта, git-состоянием и файлами репозитория.
3. **Инженерный цикл.** Инструмент подходит не только для генерации кода, но и для проверки: `git diff`, `git status`, валидация структуры, поиск устаревших ссылок.
4. **Интеграция.** В одном процессе доступны локальные файлы, терминал, браузер, MCP/tools и публикация результата в GitHub.
5. **Безопасность.** Секреты не нужно публиковать в репозитории или передавать в cloud app builder; работа идёт с локальными файлами проекта.
6. **Стоимость и доступность.** По актуальной странице OpenAI Codex pricing Codex входит в планы ChatGPT, включая Free/Go/Plus/Pro/Business/Edu/Enterprise; в текущей среде он уже доступен.

## Сравнение с альтернативами

### Cursor

**Плюсы:** сильная AI-first IDE, agentic development, rules/plugins, хороший локальный контекст.

**Минусы:** это отдельная IDE и отдельная среда. Для текущего проекта уже используется Codex, поэтому Cursor хуже отражает реальный рабочий процесс.

**Вердикт:** сильная альтернатива, но не основной инструмент для текущего проекта.

### Bolt.new, Lovable и Replit Agent

**Плюсы:** быстрый первый визуальный результат, удобны для прототипов.

**Минусы:** cloud-first workflow, слабее контроль над локальной структурой репозитория, git-историей, проверками и будущей backend/security-частью.

**Вердикт:** полезны для UI-идей и быстрых прототипов, но не как основной инструмент разработки FocusTrack AI.

### GitHub Copilot

**Плюсы:** хорошая интеграция с GitHub и IDE, сильное автодополнение, agentic features.

**Минусы:** для текущей работы меньше подходит как единая среда “анализ → правка → проверка → push”, потому что сам процесс уже ведётся в Codex.

**Вердикт:** можно использовать дополнительно, но не основной выбор.

### Cline

**Плюсы:** open-source agent runtime, Plan/Act, MCP, editor/terminal workflow.

**Минусы:** больше ручной настройки модели, бюджета и рабочего процесса.

**Вердикт:** сильная техническая альтернатива, но Codex лучше совпадает с текущей средой разработки.

## Связь с LLM-моделями из темы занятия

GigaCode, YandexGPT, DeepSeek и Qwen важны для будущего выбора модели в AI-слое FocusTrack AI. Но здесь выбирается именно рабочий **AI coding assistant** для разработки проекта. Поэтому Codex выбран как основной инструмент разработки, а модели будут оцениваться отдельно при реализации Edge Functions или другого backend-слоя.

## Вывод

Codex логично вытекает из сравнительной таблицы и из реального процесса разработки: он уже использован для работы с требованиями продукта, наведения порядка в репозитории, обновления проектных файлов, проверки структуры, коммитов и публикации результата в GitHub.
