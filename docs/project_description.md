# FocusTrack AI — описание проекта

> Трекер целей обучения и профессионального роста с AI-планировщиком  
> Курс: «ИИ-агенты: продвинутое внедрение и использование» (Otus)  
> Путь: A — одна идея через ДЗ 1→6 → проектная защита

---

## Проблема

Специалисты, которые учатся новому (курс, смена стека, pet-проект), ставят цели в заметках или todo-листах, но:

- план получается нереалистичным или слишком общим;
- нет связи между «хочу» и конкретными шагами на ближайшие недели;
- сложно понять, что сработало, а что нет — нет осмысленного review по фактам.

В итоге цели откладываются, momentum теряется.

## Решение

**FocusTrack AI** — веб-приложение, где пользователь формулирует цель обучения, проходит AI-уточнение, получает план с задачами, отмечает прогресс и раз в неделю получает **AI Review** на основе реально выполненных (и пропущенных) задач.

## Целевая аудитория

- Разработчики и IT-специалисты в процессе апскилла (новый стек, сертификация, pet-проект)
- Студенты продвинутых курсов (в т.ч. Otus), которым нужен личный трекер учебных целей
- Primary persona: «Алексей, 32 года, backend-разработчик, проходит курс по AI-агентам и параллельно ведёт pet-проект»

## Ценность продукта

| Для пользователя | Как FocusTrack AI помогает |
|------------------|-------------------------|
| Реалистичный план | AI задаёт уточняющие вопросы перед декомпозицией |
| Конкретные шаги | Цель → этапы → задачи на 2 недели с оценкой усилий (S/M/L) |
| Видимый прогресс | Дашборд: % выполнения, streak, просроченные задачи |
| Осмысленный feedback | Weekly AI Review по фактам из БД, не «из головы» модели |

## MVP — ключевые функции (Must have)

### 1. SMART-вход цели + AI Clarify

Пользователь описывает цель в свободной форме. AI (OpenRouter) задаёт 2–3 уточняющих вопроса (срок, доступное время в неделю, текущий уровень, ограничения). После ответов формируется уточнённая формулировка цели.

### 2. AI-декомпозиция (Plan)

На основе уточнённой цели AI генерирует:

- 3–5 этапов (milestones);
- список задач на ближайшие 2 недели;
- оценку усилий каждой задачи: S / M / L.

Пользователь подтверждает план и сохраняет в БД.

### 3. Дашборд прогресса

- Список активных целей с % выполнения
- Задачи на сегодня / неделю
- Отметка: done / skip / postpone
- Простая визуализация прогресса (Recharts)
- Состояния: loading, error, empty

### 4. Weekly AI Review

Раз в неделю (или по кнопке) AI анализирует:

- сколько задач выполнено / пропущено / перенесено за последние 7 дней;
- какие цели продвинулись, какие стоят.

Результат: краткий отчёт (3–5 предложений) + 3 конкретные рекомендации на следующую неделю.  
**Важно:** модель получает только факты из БД (JSON), без выдумывания.

## AI-цикл (демо на защите)

```
Создать цель → Clarify → ответы → Plan → задачи в БД
       → выполнение в течение недели → Weekly Review → рекомендации
```

## Нефункциональные требования

- Авторизация: каждый пользователь видит только свои цели (RLS)
- Секреты (OpenRouter API key) только на сервере (Edge Functions secrets)
- Адаптивный UI: desktop + mobile
- Запуск локально: `supabase start` + `npm run dev` (инструкция в README)
- LLM-ошибки: понятное сообщение пользователю, лог в `ai_sessions`
