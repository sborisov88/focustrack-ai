# Отчёт: выбор AI-инструмента для FocusTrack AI

**Проект:** FocusTrack AI
**Репозиторий:** https://github.com/sborisov88/focustrack-ai
**Выбранный инструмент:** Codex

---

## 1. Задача и требования

**FocusTrack AI** — трекер целей обучения с AI-планировщиком.

Подробные требования — в файле `requirements.md`. Коротко: это fullstack-приложение на React и Supabase с AI через OpenRouter и Edge Functions. Проект развивается несколько месяцев, поэтому нужен инструмент для локальной разработки с git, а не одноразовая генерация в облаке.

---

## 2. Сравнительный анализ

Сравнил семь инструментов:

- Cloud: **Bolt.new**, **Lovable**, **Replit Agent**;
- IDE/CLI/Desktop: **Codex**, **Cursor**, **GitHub Copilot**, **Cline**.

Полная таблица с баллами, пояснениями по каждой оценке и ссылками на актуальные источники — в файле `comparison_table.md`.

Главное наблюдение: cloud-агенты хороши для быстрого первого экрана, но хуже подходят для долгой работы с локальным репозиторием, git-историей, проверкой изменений и будущей backend/security-частью. IDE/CLI-агенты лучше подходят для такого проекта.

Отдельно зафиксирована связь с GigaCode/YandexGPT/DeepSeek/Qwen: эти решения важны как LLM-модели или провайдеры для будущего AI-слоя, но не заменяют основной coding-agent workflow для разработки проекта.

---

## 3. Почему выбран Codex

Подробное обоснование и разбор альтернатив — в файле `tool_selection.md`.

FocusTrack AI — проект на несколько месяцев, где важны локальный репозиторий, git, контролируемые изменения, терминал, MCP/tools и публикация результата в GitHub. Codex и Cursor по этим возможностям находятся на одном уровне:

- работа с локальными файлами проекта;
- обновление проектной документации;
- генерация и правка TypeScript/React-кода;
- проверка `git diff`, `git status` и состава репозитория;
- коммиты и push в GitHub.

Решающий плюс Codex — стоимость. По OpenAI Codex pricing он включён в планы ChatGPT, включая Free/Go/Plus/Pro/Business/Edu/Enterprise. Cursor по своей pricing-странице остаётся отдельной подпиской с включённым объёмом model usage и возможной оплатой on-demand usage после исчерпания включённого объёма.

В таблице Codex набирает 4,8 против 4,5 у Cursor. Разрыв появляется только из-за стоимости: по контексту, скорости, качеству кода, безопасности и интеграции инструменты оценены одинаково.

## 6. Выводы

1. Для fullstack-проекта на несколько месяцев нужен IDE/CLI/Desktop-агент, а не cloud-конструктор первого прототипа.
2. **Codex** — основной инструмент для FocusTrack AI: он обеспечивает полный цикл работы с локальным проектом и при этом уже включён в подписку ChatGPT.
3. Cursor остаётся сильной альтернативой и находится на одном уровне с Codex по ключевым возможностям, но требует отдельной подписки.
4. Cloud-агенты можно использовать точечно для UI-идей, но не как основной engineering workflow.
5. Следующий шаг — правила и промпт-шаблоны под Codex workflow.

## 7. Фиксация для следующих этапов

Дальнейшие этапы FocusTrack AI выполняются в **Codex**, а не в Cursor.

Для следующего этапа это означает:

- постоянные правила проекта оформляются как `AGENTS.md`, потому что это официальный механизм проектных инструкций Codex;
- описание правил хранится в `docs/prompts/rules.md`;
- `.cursorrules` не создаётся, так как Cursor не является выбранным рабочим инструментом проекта;
- Cursor остаётся только сравнительной альтернативой.
