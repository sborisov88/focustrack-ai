## Рекомендация курса: одна идея на весь цикл

**Совет курса (занятие 28):** закрепить материал через **единый консистентный проект**. Удобный способ — вести **одну идею** через ДЗ 1→6, чтобы к защите накопился репозиторий и MVP.

> **Важно:** проект и домашние задания — **разные работы** на платформе Otus (отдельные чаты, отдельные оценки). Проект — это не «автоматическая сумма» шести ДЗ, а **самостоятельная защита MVP** (~10 мин), где вы показываете применение навыков курса (см. Q&A занятия 28: *«не просто сдаёт отдельные домашние задания»*).

### Что переносится из ДЗ в проект, а что нет

| Из ДЗ | Переносится в проект? | Как используется |
|-------|----------------------|------------------|
| ДЗ 1 (сравнение инструментов) | **Частично** | В презентации: обоснование выбора Cursor/IDE; отчёт не входит в код приложения |
| ДЗ 2 (rules, промпты) | **Частично** | Процесс разработки; на защите — 2–3 ключевых промпта |
| ДЗ 3 (ТЗ, UI, stories) | **Да** | Основа продукта: требования, дизайн, сценарии |
| ДЗ 4 (frontend) | **Да** | Часть MVP |
| ДЗ 5 (backend, e2e) | **Да** | Часть MVP |
| ДЗ 6 (CI/CD, OAuth, security) | **Желательно** | Плюс к проекту; для самого проекта не строго обязательно (Q&A: нет жёсткого отказа) |

### Два пути к проекту (оба допустимы)

| Путь | Описание |
|------|----------|
| **A — Последовательный (рекомендуется курсом)** | Одна идея в ДЗ 1→6 → проект = **доработка MVP + презентация + защита**. Пример: Рустам «связывал лабораторные работы с финальным проектом» (занятие 29) |
| **B — Отдельный проект** | ДЗ сдаются как учебные артефакты; проект — **другая или расширенная тема**, но те же компетенции. Явно допустимо гибкими требованиями занятия 28 |

### Чего проект требует сверх ДЗ

- Утверждённая **тема** в чате проектного задания
- **Защита** ~10 минут с демо
- Показать **применение** инструментов курса, а не только папку с отчётами ДЗ
- Желательно **AI/LLM в продукте** (RAG, агент, классификатор) — для проекта важнее, чем для отдельных ДЗ

## Карта связей (полная)

```
ДЗ 1 (инструмент)
  ↓  выбор Cursor/IDE, обоснование
ДЗ 2 (rules + промпты)
  ↓  .cursor/rules/, prompt_templates.md
ДЗ 3 (ТЗ + UI + stories)
  ↓  technical_specification.md, ui_concepts/, user_stories.md
ДЗ 4 (frontend, mock-данные)
  ↓  ← обязательно: ТЗ из ДЗ 3, rules из ДЗ 2
ДЗ 5 (backend + БД + e2e)
  ↓  ← frontend из ДЗ 4, требования из ДЗ 3
ДЗ 6 (CI/CD + OAuth + аналитика + security)
  ↓  ← приложение из ДЗ 4–5
ПРОЕКТ (защита MVP ~10 мин)
     ← может опираться на ДЗ 3–6; ДЗ 1–2 — в основном для презентации
```

```
┌─────────────────────────────────────────────────────────────────┐
│                     ПРОЕКТНАЯ РАБОТА (финал)                     │
│  Тема + MVP + защита + репозиторий + презентация                │
└───────────────────────────────┬─────────────────────────────────┘
                                │ собирает артефакты из:
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
   ДЗ 1: инструмент      ДЗ 2: rules/промпты     ДЗ 3: ТЗ/UI/stories
        │                       │                       │
        └───────────────────────┼───────────────────────┘
                                ▼
                    ДЗ 4: Frontend (mock → UI)
                                │
                                ▼
                    ДЗ 5: Backend + БД + e2e
                                │
                                ▼
                    ДЗ 6: CI/CD + OAuth + аналитика + security
```

## Зависимости между ДЗ (что откуда брать)

| ДЗ | Берёт из предыдущих | Отдаёт следующим |
|----|---------------------|------------------|
| **1** | — | Идея проекта, выбранный AI-агент |
| **2** | Идея и агент из ДЗ 1 | `rules.md`, `prompt_templates.md` → **ДЗ 4** |
| **3** | Идея из ДЗ 1–2 | `technical_specification.md`, UI, stories → **ДЗ 4, 5, 6** |
| **4** | **ДЗ 2:** rules; **ДЗ 3:** ТЗ, UI | Frontend с mock → **ДЗ 5** |
| **5** | **ДЗ 3:** требования к данным; **ДЗ 4:** frontend | API, БД, e2e → **ДЗ 6** |
| **6** | **ДЗ 4–5:** приложение | CI/CD, OAuth, security → **Проект** |
| **Проект** | Артефакты ДЗ 1–6 | Презентация, демо, README |

### Жёсткие ссылки в формулировках заданий

- **ДЗ 4:** «использовать ТЗ из ДЗ 3», «применить `.cursorrules` из ДЗ 2», «промпт-шаблоны из ДЗ 2»
- **ДЗ 5:** frontend из ДЗ 4 заменить mock на API
- **ДЗ 6:** деплой и тесты приложения из ДЗ 4–5
