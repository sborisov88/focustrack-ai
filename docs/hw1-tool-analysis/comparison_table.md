## Bolt.new (Cloud)

**Сильные стороны:** очень быстро даёт работающий UI в браузере; удобен, когда нужно «показать экран» за час.

**Слабые стороны для FocusTrack AI:** проект живёт в экосистеме вендора; сложно вести Supabase migrations, RLS и Edge Functions как полноценный monorepo; риск vendor lock-in; не подходит для длительной brownfield-разработки.

**Итог:** хорош для разового прототипа, не как основной инструмент на весь курс.

---

## Lovable (Cloud)

**Сильные стороны:** похож на Bolt — быстрый визуальный результат, мало порога входа.

**Слабые стороны для FocusTrack AI:** те же ограничения cloud-агентов — слабый контроль над файловой структурой, git-процессом и backend-слоем; неудобно хранить секреты и серверную логику.

**Итог:** можно рассмотреть только для UI-концепций, но не для fullstack MVP.

---

## Replit Agent (Cloud)

**Сильные стороны:** быстрый старт, облачная среда, можно сразу запускать приложение.

**Слабые стороны для FocusTrack AI:** проект курса уже ведётся как локальный git-репозиторий; Supabase migrations, RLS и Edge Functions удобнее контролировать в IDE.

**Итог:** сильнее Bolt/Lovable для полноценного проекта, но всё равно уступает IDE-агенту как основному инструменту курса.

---

## Cursor (IDE)

**Сильные стороны:** полный доступ к локальному репозиторию; Agent Mode и Composer; `.cursor/rules`; MCP (документация, браузер и др.); удобен для React + Supabase + документации в одном проекте.

**Слабые стороны:** нужна дисциплина в промптах; иногда агент добавляет лишнее — исправляется уточняющими запросами.

**Итог:** лучший выбор для FocusTrack AI на весь цикл курса.

---

## GitHub Copilot (IDE)

**Сильные стороны:** хорошее автодополнение в VS Code; привычный редактор; разумная стоимость.

**Слабые стороны для FocusTrack AI:** слабее как автономный агент: нет того же уровня Agent Mode, Rules и MCP «из коробки», что у Cursor; больше подходит как помощник при написании кода, чем для архитектуры, ADR и настройки monorepo.

**Итог:** сильный дополнительный инструмент, но не основной для этого проекта.

---

## Cline (IDE/CLI)

**Сильные стороны:** хороший локальный агент, умеет работать с файлами проекта, подходит для точечных правок и code review.

**Слабые стороны для FocusTrack AI:** требует больше ручной настройки workflow; для ДЗ 2 Cursor удобнее, потому что rules, агентный режим и IDE-интерфейс находятся в одном месте.

**Итог:** достойная альтернатива, но не лучший основной инструмент на весь учебный цикл.

## Отдельно: LLM-модели из темы занятия

GigaCode, YandexGPT, DeepSeek и Qwen важны для выбора модели и провайдера, но в этом ДЗ основной объект сравнения — **AI coding assistant** как рабочий инструмент разработки. Для FocusTrack AI они рассматриваются как возможный LLM-слой:

| Модель / направление | Роль в проекте | Вывод для ДЗ 1 |
|----------------------|----------------|----------------|
| GigaCode / GigaChat | Российская экосистема, возможна для корпоративных сценариев | Не основной coding-agent интерфейс для этого курса |
| YandexGPT | Российский LLM-провайдер | Можно учитывать для будущего AI-слоя, но не заменяет IDE-агента |
| DeepSeek | Open-source / доступные модели для reasoning и code | Кандидат для вызовов через OpenRouter после отдельной оценки качества |
| Qwen | Open-source семейство моделей | Кандидат для on-premise или OpenRouter-сценариев, не основной инструмент разработки |

Итог: для **разработки проекта** выбран Cursor; для **продуктового AI-слоя** модели будут оцениваться отдельно при реализации Supabase Edge Functions.

## Общий вывод по таблице

Cloud-агенты выигрывают по скорости первого экрана, но проигрывают по контексту, безопасности и долгосрочной разработке.

Среди IDE-агентов **Cursor** набирает наивысший балл именно под задачу FocusTrack AI.
