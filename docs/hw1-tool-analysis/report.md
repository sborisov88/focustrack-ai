## 1. Задача и требования

Я выбрал одну идею на весь курс — **FocusTrack AI**, трекер целей обучения с AI-планировщиком.

Подробные требования — в файле `requirements.md`. Коротко: это fullstack-приложение на React и Supabase с AI через OpenRouter и Edge Functions. Проект развивается несколько месяцев, поэтому нужен инструмент для локальной разработки с git, а не одноразовая генерация в облаке.

---

## 2. Сравнительный анализ

Сравнил семь инструментов:

- Cloud: **Bolt.new**, **Lovable**, **Replit Agent**;
- IDE/CLI/Desktop: **Codex**, **Cursor**, **GitHub Copilot**, **Cline**.

Полная таблица с баллами, пояснениями по каждой оценке и ссылками на актуальные источники — в файле `comparison_table.md`.

Главное наблюдение: cloud-агенты хороши для быстрого первого экрана, но хуже подходят для долгой работы с локальным репозиторием, git-историей, проверкой артефактов и будущей backend/security-частью. IDE/CLI-агенты лучше подходят для такого проекта.

Отдельно зафиксировал связь с темой занятия про GigaCode/YandexGPT/DeepSeek/Qwen: эти решения важны как LLM-модели или провайдеры для будущего AI-слоя, но не заменяют основной coding-agent workflow для разработки проекта.

## 3. Почему выбран Codex

Подробное обоснование и разбор альтернатив — в файле `tool_selection.md`.

Кратко: FocusTrack AI — проект на несколько месяцев, где важны локальный репозиторий, git, проверяемые изменения, терминал, браузер, MCP/tools и публикация результата в GitHub. Codex уже использован в этом ДЗ для полного цикла:

- прочитать обычное и расширенное ДЗ;
- сверить требования с репозиторием;
- обновить markdown-артефакты;
- проверить структуру и отсутствие лишних файлов;
- сделать коммиты и push в GitHub.

В таблице Codex набирает 4,8 против 4,7 у Cursor, 4,2 у GitHub Copilot, 3,8 у Cline и 2,7–3,2 у cloud-инструментов.

## 6. Выводы

1. Для учебного fullstack-проекта на несколько месяцев нужен IDE/CLI/Desktop-агент, а не cloud-конструктор первого прототипа.
2. **Codex** — основной инструмент для FocusTrack AI, потому что он уже обеспечивает полный цикл работы: анализ, правки, проверки, git, коммиты и push.
3. Cursor остаётся сильной альтернативой, но хуже отражает фактическую среду выполнения этого ДЗ.
4. Cloud-агенты можно использовать точечно для UI-идей, но не как основной engineering workflow.
5. Следующий шаг — ДЗ 2: правила и промпт-шаблоны уже стоит писать под Codex workflow.
