# FocusTrack AI — требования для выбора AI-инструмента

## Задача

Разработать **FocusTrack AI** — fullstack веб-приложение для отслеживания целей обучения и профессионального роста с AI-циклом: уточнение цели → план → выполнение задач → еженедельный обзор прогресса.

Проект ведётся на протяжении всего курса: от документации и UI до backend, интеграции и деплоя.

## Ключевые требования

**Сложность проекта.** Не лендинг и не прототип на один вечер. Нужен полноценный стек: React + TypeScript, Supabase (PostgreSQL, Auth, RLS), Edge Functions для вызова OpenRouter, CI/CD, OAuth, аналитика.

**Работа с локальными файлами.** Код, миграции БД, Edge Functions, документация и правила для агента должны жить в одном репозитории. Важны git, diff и возможность точечно править результат.

**Безопасность.** API-ключ OpenRouter нельзя держать на клиенте. Пользовательские данные изолируются через RLS. Секреты — только в Supabase secrets.

**Бюджет.** Проект рассчитан на несколько месяцев. Инструмент с оплатой «за каждую генерацию в облаке» неудобен как основной. Нужна предсказуемая подписка на IDE-агент.

**Тип разработки.** Brownfield: проект развивается итерациями, а не генерируется один раз «с нуля в браузере».

## MVP (что должно получиться)

1. SMART-вход цели с AI-уточнением
2. AI-декомпозиция цели в этапы и задачи
3. Дашборд прогресса
4. Weekly AI Review по фактическим данным из БД

## Вывод для выбора инструмента

Нужен **AI coding assistant** с доступом ко всему локальному репозиторию, git, терминалу, проверкам и MCP/tools — не cloud-конструктор для быстрого MVP. По итогам сравнения для ДЗ 1 выбран Codex.
