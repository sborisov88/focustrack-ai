# Обоснование выбора AI-инструмента

## Выбранный инструмент

**Cursor** — основной IDE-агент на весь курс и проектную работу.

## Ключевые факторы решения

1. **Тип проекта.** FocusTrack AI — brownfield fullstack monorepo, а не одноразовый MVP в браузере.
2. **Контекст.** Нужен доступ ко всему дереву: `frontend/`, `supabase/`, `docs/`, миграции, Edge Functions.
3. **Безопасность.** Секреты OpenRouter только на сервере; cloud-агенты не дают нужного контроля.
4. **Длительность.** Проект на 2–3 месяца — важны git, diff, Rules, MCP.
5. **Следующие ДЗ.** ДЗ 2 требует `.cursor/rules` — нативно поддерживается в Cursor.

## Сравнение с альтернативами

### Bolt.new, Lovable и Replit Agent (Cloud)

**Плюсы:** очень быстрый первый экран.
**Минусы:** vendor lock-in, слабый backend-слой, менее удобная работа с Supabase migrations/RLS и локальным git-flow.
**Вердикт:** не подходят как основной инструмент; допустимы только для UI-мока в ДЗ 3.

### GitHub Copilot (IDE)

**Плюсы:** хорошее автодополнение, знакомый VS Code.
**Минусы:** слабее как автономный агент — нет Agent Mode, Rules и MCP на том же уровне.
**Вердикт:** помощник при написании кода, не замена Cursor для архитектуры и документации.

### Cline (CLI)

**Плюсы:** сильный агент, работа с файлами.
**Минусы:** меньше цельной IDE-интеграции, чем у Cursor; Rules и MCP настраиваются иначе.
**Вердикт:** рассмотрен, но Cursor удобнее для единого workflow на весь курс.

## Связь с LLM-моделями из темы занятия

GigaCode, YandexGPT, DeepSeek и Qwen важны для будущего выбора модели в AI-слое FocusTrack AI. Но в ДЗ 1 выбирается именно рабочий **AI coding assistant** для разработки проекта. Поэтому Cursor выбран как основной инструмент разработки, а модели будут оцениваться отдельно при реализации Edge Functions через OpenRouter.
