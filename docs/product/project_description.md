# FocusTrack AI — описание продукта

## Идея

FocusTrack AI — веб-приложение для людей, которые ведут длинные личные и рабочие цели и теряют фокус между повседневными делами, работой и долгосрочными планами.

Продукт помогает превратить размытую цель в проверяемый план: уточнить формулировку, разложить цель на задачи, отслеживать прогресс и получать еженедельный AI-обзор по фактам выполнения.

## Проблема

У пользователя часто есть цель вроде «пробежать первый полумарафон», «сдать IELTS на 7.0», «собрать финансовую подушку» или «запустить лендинг пет-проекта», но нет устойчивого цикла:

- что именно надо сделать на этой неделе;
- какие задачи уже закрыты;
- где появился блокер;
- какие действия реально продвигают цель;
- как объективно оценить собственный прогресс.

Обычный список задач не помогает с уточнением цели и регулярной рефлексией. Чат с моделью помогает разово, но не хранит структуру прогресса. FocusTrack AI объединяет эти два мира.

## Целевая аудитория

- люди, которые готовятся к спортивной цели (например, первый полумарафон);
- те, кто готовится к экзамену или сертификации (например, IELTS);
- люди, которые ведут личный финансовый план (например, подушка безопасности);
- авторы пет-проектов, которым нужно довести идею до запуска.

## Ценность

Пользователь получает один рабочий контур:

1. вводит цель и контекст;
2. получает уточняющие вопросы от AI;
3. превращает цель в задачи;
4. отмечает выполнение;
5. видит прогресс и блокеры;
6. получает weekly review с рекомендацией следующего шага.

## MVP-функции

### 1. SMART-вход цели

Пользователь создаёт цель, описывает контекст, дедлайн и ограничения. AI помогает уточнить цель через вопросы, чтобы формулировка стала проверяемой.

### 2. AI-декомпозиция

После уточнения цель раскладывается на этапы и задачи. В MVP план ограничен ближайшими двумя неделями, чтобы не создавать ложную точность.

### 3. Дашборд прогресса

Пользователь видит цели, задачи, статусы, процент выполнения, блокеры и личные заметки. Интерфейс реализован на React, TypeScript, shadcn/ui, Tailwind и Recharts.

### 4. Weekly AI Review

Раз в неделю пользователь запускает AI-обзор. Серверная функция отправляет в модель только факты прогресса: выполненные задачи, блокеры и процент выполнения.

### 5. Ответы по личным заметкам (RAG)

Пользователь может задать вопрос по своим заметкам и истории прогресса, а модель отвечает только по переданному контексту.

## Границы MVP

Входит в MVP:

- локальный и облачный Supabase-контур;
- миграции PostgreSQL;
- RLS-политики;
- Edge Functions для AI;
- OpenRouter только со стороны сервера;
- Playwright-проверка с видео и скриншотами.

Не входит в первый MVP:

- сложная командная работа;
- платёжная система;
- мобильное приложение;
- полноценный RAG-индекс с эмбеддингами.

## Выбранный стек

| Слой       | Решение                                                |
| ---------- | ------------------------------------------------------ |
| Frontend   | React, TypeScript, Vite, shadcn/ui, Tailwind, Recharts |
| State/data | React Query, Supabase client                           |
| Backend    | Supabase Cloud                                         |
| DB         | PostgreSQL + RLS                                        |
| AI         | Supabase Edge Functions + OpenRouter                   |
| Тесты      | Vitest, Playwright                                     |
| CI/CD      | GitHub Actions + опциональный Supabase deploy          |

## Критерии успешного MVP

- приложение запускается локально;
- дашборд показывает цели, задачи, прогресс и weekly review;
- можно создать новую цель и отметить задачу выполненной;
- Supabase-миграции применяются локально и в cloud;
- Edge Function вызывает OpenRouter с серверным секретом;
- Playwright сохраняет скриншоты и видео основного сценария;
- продукт задеплоен в production на Vercel.
