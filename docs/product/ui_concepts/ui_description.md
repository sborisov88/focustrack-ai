# UI-концепции FocusTrack AI

## Метод

Концепции сформулированы как text-to-UI задания для AI-агента и затем сведены в реализуемый интерфейс на shadcn/ui. Цель — выбрать не красивую витрину, а рабочий экран, который помогает быстро проверять цели, задачи, прогресс и AI-сессии.

## Концепция 1 — Minimal Focus

**Стиль:** спокойный минималистичный интерфейс для индивидуальной работы.

**Компоновка:**

- верхняя панель с названием продукта и быстрым созданием цели;
- список целей слева;
- детали выбранной цели в центре;
- AI Review и сводка по цели справа.

**Плюсы:**

- не перегружает пользователя;
- хорошо подходит для ежедневного использования;
- проще адаптируется под мобильный экран.

**Минусы:**

- меньше аналитической плотности;
- нужен отдельный блок для истории AI-сессий.

## Концепция 2 — Dashboard Heavy

**Стиль:** плотный рабочий дашборд для проверки статуса всех целей сразу.

**Компоновка:**

- sidebar с навигацией;
- карточки целей и метрик;
- таблица AI-сессий;
- график прогресса;
- Knowledge/RAG-блок.

**Плюсы:**

- хорошо показывает прогресс по нескольким целям сразу;
- удобно демонстрировать продукт целиком;
- пользователь быстро видит состояние всех целей.

**Минусы:**

- выше риск перегрузить первый экран;
- требует аккуратной адаптивной сетки.

## Концепция 3 — Mobile Coach

**Стиль:** mobile-first помощник, похожий на персонального тренера по цели.

**Компоновка:**

- одна текущая цель;
- крупный следующий шаг;
- короткий список задач;
- кнопка weekly review;
- история рекомендаций ниже.

**Плюсы:**

- лучший сценарий для ежедневного использования;
- минимальное когнитивное усилие;
- хорошо подходит для push-уведомлений в будущем.

**Минусы:**

- хуже подходит для демонстрации всей архитектуры;
- сложнее показать несколько целей и backend-артефактов одновременно.

## Выбор

Для MVP выбрана гибридная версия **Dashboard Heavy + Minimal Focus**:

- shadcn Sidebar даёт навигацию и инфраструктурный статус;
- карточки и таблица показывают цели, задачи и AI-сессии;
- центральная зона остаётся сфокусированной на выбранной цели;
- мобильный viewport сохраняет основной контент и кнопку создания цели.

Такой вариант лучше всего соответствует задаче продукта: приложение должно быть не лендингом, а рабочим инструментом для ведения целей.

## Реализация

Выбранная концепция реализована в:

- `src/features/dashboard/focustrack-dashboard.tsx`;
- `src/components/ui/*` — shadcn/ui;
- `src/lib/demo-data.ts` — демонстрационные данные;
- `output/playwright/screenshots/` — скриншоты после запуска Playwright.
