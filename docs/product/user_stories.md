# User Story Mapping

## Персоны

| Персона      | Контекст                         | Главная потребность                               |
| ------------ | -------------------------------- | ------------------------------------------------- |
| Студент OTUS | Ведёт ДЗ и проект                | Понимать следующий шаг и не терять критерии сдачи |
| Разработчик  | Делает pet project для портфолио | Превратить идею в работающий MVP                  |
| Ментор       | Проверяет прогресс ученика       | Видеть факты выполнения и блокеры                 |

## Карта пользовательского пути

```text
Создать цель
  -> уточнить через AI
  -> получить план
  -> выполнить задачи
  -> увидеть прогресс
  -> получить weekly review
  -> подготовить отчет/демо
```

## Must have

### US-01 — Создание цели

Как студент, я хочу создать цель с описанием и дедлайном, чтобы зафиксировать ожидаемый результат.

**Acceptance Criteria**

- Given открыт дашборд, When пользователь нажимает «Новая цель», Then появляется форма создания цели.
- Given название короче 3 символов, When пользователь пытается отправить форму, Then кнопка отправки недоступна.
- Given валидная цель заполнена, When пользователь нажимает «Добавить», Then цель появляется в списке.

### US-02 — AI-уточнение цели

Как студент, я хочу получить уточняющие вопросы от AI, чтобы цель стала проверяемой.

**Acceptance Criteria**

- Given у цели есть описание, When запускается AI-clarify, Then пользователь получает список уточняющих вопросов.
- Given OpenRouter временно недоступен, When функция возвращает ошибку, Then пользователь видит понятное сообщение без технических секретов.
- Given ответ AI получен, When данные сохраняются, Then сессия появляется в журнале `ai_sessions`.

### US-03 — План задач

Как разработчик, я хочу получить план задач на ближайшие две недели, чтобы двигаться к MVP маленькими шагами.

**Acceptance Criteria**

- Given цель уточнена, When запускается AI-plan, Then создаётся список задач с due date, effort и статусом.
- Given AI вернул неполный план, When функция валидирует ответ, Then пользователь получает ошибку валидации.
- Given задачи созданы, When пользователь открывает цель, Then задачи отображаются в порядке `sort_order`.

### US-04 — Отметка выполнения и прогресс

Как студент, я хочу отмечать задачи выполненными, чтобы видеть реальный процент прогресса.

**Acceptance Criteria**

- Given задача находится в статусе `todo` или `doing`, When пользователь отмечает чекбокс, Then статус становится `done`.
- Given статус задачи изменился, When пересчитывается цель, Then процент прогресса обновляется.
- Given задача снята с выполнения, When пользователь убирает чекбокс, Then прогресс уменьшается.

### US-05 — Weekly AI Review

Как студент, я хочу получить еженедельный обзор по фактам выполнения, чтобы выбрать следующий шаг.

**Acceptance Criteria**

- Given есть выполненные и заблокированные задачи, When пользователь нажимает `AI Review`, Then Edge Function возвращает краткий обзор.
- Given OpenRouter ключ хранится в Supabase secrets, When frontend вызывает review, Then ключ не появляется в браузере.
- Given review получен, When сессия завершена, Then результат отображается в панели AI Review.

## Should have

### US-06 — OAuth-вход

Как пользователь, я хочу войти через Google, чтобы не создавать отдельный пароль.

**Acceptance Criteria**

- Given Supabase Auth настроен, When пользователь нажимает `Google`, Then начинается OAuth flow.
- Given provider не настроен, When Supabase возвращает ошибку, Then пользователь видит toast с безопасным сообщением.

### US-07 — Ответы по документам

Как студент, я хочу задать вопрос по документам проекта, чтобы быстро найти требования к сдаче.

**Acceptance Criteria**

- Given в запрос переданы документы, When вызывается `rag-answer`, Then ответ строится только по переданному контексту.
- Given данных недостаточно, When модель не может ответить, Then она явно сообщает о нехватке данных.

## Nice to have

- экспорт weekly review в Markdown;
- push-напоминания о следующем шаге;
- командный режим для ментора;
- полноценный RAG-индекс на pgvector.

## Edge Cases

| Сценарий                | UX-решение                                              |
| ----------------------- | ------------------------------------------------------- |
| Пустой список целей     | Показать демо-цель и кнопку создания                    |
| Сетевая ошибка Supabase | Сохранить демо-режим и показать toast                   |
| Ошибка OpenRouter       | Показать безопасное описание и разрешить повтор         |
| Нет авторизации         | Не загружать приватные данные, показывать демо-данные   |
| Длинный ответ AI        | Ограничить блок review, сохранить полный текст в сессии |
