# FocusTrack AI — структура защиты

## Тайминг на 10 минут

| Блок | Время |
| --- | --- |
| Проблема и решение | 1:30 |
| Демо продукта | 3:00 |
| Архитектура и безопасность | 2:00 |
| AI workflow и роль автора | 1:30 |
| Сложности, ограничения и следующие шаги | 2:00 |

## 1. Проблема

Человеку сложно удерживать фокус на длинной личной или рабочей цели (полумарафон, IELTS, финансовая подушка, запуск пет-проекта): уточнение цели, план задач, прогресс, weekly review и личные заметки живут в разных местах.

## 2. Решение

FocusTrack AI — AI-планировщик личных и рабочих целей: объединяет цель, AI-уточнение, AI-план задач, прогресс, еженедельное AI-ревью и ответы по личным заметкам (RAG) в одном рабочем интерфейсе.

## 3. Демо-сценарий (на жизненной цели)

1. Открыть дашборд: «Категории целей» и «Личный планировщик».
2. Создать цель «Пробежать первый полумарафон».
3. Пройти AI-уточнение и получить AI-план задач.
4. Отметить задачу выполненной и показать пересчет прогресса.
5. Запустить еженедельное AI-ревью (weekly review).
6. Задать RAG-вопрос по личным заметкам: «на какой неделе была самая длинная пробежка».
7. Показать Supabase schema, RLS и Edge Functions.
8. Показать Playwright screenshots/video.

## 4. Архитектура

```text
React + shadcn
  -> Supabase client
  -> PostgreSQL + RLS
  -> Edge Functions
  -> OpenRouter
```

## 5. Что показать из курса (освоенные навыки)

- выбор Codex как AI coding assistant;
- AGENTS.md и prompt templates;
- ДЗ 3: ТЗ, user stories, UI-концепции;
- ДЗ 4: frontend;
- ДЗ 5: backend;
- ДЗ 6: CI/CD, OAuth, analytics, security;
- проект: рабочий MVP AI-планировщика личных и рабочих целей (сквозной сценарий на жизненной цели + RAG по личным заметкам).

## 6. Роль AI и роль автора

| Зона | AI | Автор |
| --- | --- | --- |
| Идея продукта | помог структурировать варианты MVP и пользовательские сценарии | выбрал тему, границы MVP и жизненные примеры целей |
| Проектирование | помог разложить требования на user stories, ТЗ и архитектурные решения | проверил реализуемость, выбрал стек и зафиксировал ADR |
| Реализация | ускорил генерацию React, Supabase Functions, тестов и документации | принимал архитектурные решения, проверял код, запускал тесты и деплой |
| Проверка | помог составить e2e-сценарии и acceptance matrix | провел локальные и production-проверки, устранил несоответствия |

## 7. Сложности

- связать demo-mode и Supabase session-mode без раздвоения пользовательского интерфейса;
- защитить AI/RAG функции JWT и не раскрывать OpenRouter ключ на клиенте;
- сделать длинный AI-clarify flow удобным в модальном окне на desktop и mobile;
- синхронизировать документы, evidence и фактическую реализацию после нескольких итераций.

## 8. Что не входит в MVP

- совместная работа нескольких пользователей в одном workspace;
- полноценный календарь и push-уведомления;
- платные тарифы и биллинг;
- production rate limit на AI-вызовы;
- расширенная аналитика привычек и долгосрочных трендов.
